what are we doing?

- following 0BAB1's project to implement RISC-V RV32I onto an fpga and hopefully port a game to it

- the project states:
	we will be using SystemVerilog for the underlying logic of the CPU
	we will use DDCA lectures chapter 7
	we will be using Verilator and Cocotb to test 

- project suggestions:
	use linux (i will be using windows subsystem for linux (WSL))
	know basics of Python, RISC-V assembly, and SystemVerilog + digital design

- to activate the test environment (personal use):
	1. type wsl into powershell
	2. go to the home directory
	3. use the "source" command 'source venv/bin/activate'
	4. make a Makefile and specify simulation type and language and use cocotb rules for makefile
	
- start goal:
	1. implment "lw," an I-type (immediate type)
		cpu instruction example "lw x6, -4(x9)"
	lw -> 010 (f3, instruction)
	x6 00110 (rd, a register)
	-4 -> 111111111100 (IMM, or immediate)
	x9 -> 01001 (rs1, a register)
	I-type -> 0000011 (op-code, or operation type)
